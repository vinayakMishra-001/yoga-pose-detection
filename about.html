About This Project

About YogaVision

A computer vision project combining real-time body landmark detection with rule-based pose classification — built entirely in Python.

What Is YogaVision?

YogaVision is an AI-powered yoga pose detection system that uses Google's MediaPipe framework and OpenCV to analyse human body posture in real time. It detects 33 skeletal landmarks and classifies yoga poses by measuring joint angles against predefined thresholds.

The project demonstrates how computer vision and geometric mathematics can be applied to a health and wellness use case — making pose feedback accessible without specialised hardware or apps.

Project Concept & Research

Identified MediaPipe as the ideal framework for real-time body landmark detection. Researched existing pose estimation approaches and defined the project scope: a lightweight, angle-based classifier running entirely in Python without deep learning training overhead.

🔧
BUILD

Core Detection Engine

Built the calculate_angle() function using NumPy's arctan2 to compute joint angles from 3D landmark coordinates. Integrated OpenCV for video capture, frame processing, and visual overlay rendering.

🎯
CALIBRATE

Pose Threshold Calibration

Manually calibrated angle thresholds for each pose by testing across multiple subjects and camera angles. Tuned ranges to balance sensitivity (catching correct poses) against specificity (avoiding false positives).

✅
COMPLETE

Testing & Accuracy Measurement

Achieved 94% accuracy on Warrior II, 91% on Tree Pose, and 96% on T Pose through systematic testing. Built this documentation website to showcase the system architecture and make the project accessible to others.

Technology Choices

🤖 Why MediaPipe?

Google's MediaPipe Pose (BlazePose) offers state-of-the-art landmark detection optimised for real-time performance. It runs at 30–60fps on consumer hardware, outputs 33 3D landmarks with confidence scores, and requires no GPU or specialised hardware — making it ideal for accessible, lightweight pose estimation without training a custom model.

👁️ Why OpenCV?

OpenCV is the industry standard for computer vision tasks in Python. It handles video capture from webcams and files, frame-by-frame processing, and drawing overlays (skeleton lines, text labels, angle readouts) directly on frames. Its tight integration with NumPy makes it perfect for this pipeline.

📐 Why Rule-Based Classification?

Rather than training a neural network classifier (which requires labelled datasets, training time, and hardware), YogaVision uses deterministic angle thresholds. This approach is fully explainable — each decision can be traced to exact angle values — making it easy to debug, tune, and extend with new poses.

📊 Why NumPy + Matplotlib?

NumPy provides the fast array mathematics needed for arctangent calculations across landmark coordinate pairs. Matplotlib is used for optional static analysis — plotting angle distributions, generating accuracy graphs, and visualising pose data outside of the live video loop.

⚡ Challenges & Solutions
  • 🔁
    Mirror Effect: Camera image is horizontally flipped, causing left/right landmark confusion. Solution: Applied cv2.flip() to mirror the frame before processing, aligning visual output with natural movement.
  • 🌗
    Lighting Sensitivity: Poor lighting reduces landmark confidence scores below the detection threshold. Solution: Added confidence thresholds and "Low Visibility" warnings when landmarks drop below 0.5 confidence.
  • 📦
    Partial Occlusion: Poses near frame edges cause landmarks to be clipped. Solution: Implemented bounding box checks that warn when key landmarks are near frame edges, prompting the user to step back.
  • 🔧
    Warrior II vs T-Pose Overlap: Both poses have extended arms at shoulder height. Solution: Added knee angle as a distinguishing condition — Warrior II requires one bent knee (90–120°), while T-Pose requires both legs straight (160–195°).
